FAQ

What are these starters I keep seeing?

Each starter represents an empty template to begin with, having the components in the name present. 

How do I install the item?

Copy and paste all the items in the code folder to your server to your desired location. 

What are those i class"fa " things?

That is Font Awesome, the amazing icons we used for this item you can learn more about them and find a cheatsheet on how to use them here http://fortawesome.github.io/Font-Awesome/icons/

My 404, coming soon, pageapp-login, pageapp-register, pageapp-coverpage and other pages with fullscreen content and sldiers aren't working?

Be sure to have the body set to has cover <body class="has-cover">. it's covered in the page docmentation. This optimizez JS speed and removes unnecessary handlers on pages. 

How do I change the contact form information?

There is only one PHP file in the item, please open that item and replace the email address with yours. 

How do I make the login page / sign in page / calendar page / news / store / blog  to work? 

Please remember that this is a site template and is not connected to any CMS or any databases, so these features are mearly template designs, they are HTML and CSS and need to be connected to a back-end to work.

The header is comprised of 4 icons and a logo. Let's get to explaining what each of them does. 

 
Line 1 is the header class. Notice it has a secondary class called header-dark. You can replace this with header-light to change the color of the header from dark to light. 
 
Line 2 is the first header icon from the left. The secondary class open-left-sidebar makes this icon open the left sidebar.
 
Line 3 is the second icon from the left. The secondary class  open-modal-menu makes this icon open the modal menu
 
Line 4 is the page logo. The header logo depends on the color of the header.  To change the image in the logo, open menu.css and locate line 18 ( for the dark header ) and line 25 (for the light header ) After replacing the images there with your images head over to line 51. That will be where you have to set the size of your logo by replacing the background-size, widht and height attributes with those of your logo. Make sure these are half the size of your actual image to make the image look crisp on high definition displays ( retina quality ) 
 
Line 5 is the second icon from the right . The secondary class open-header-menu makes this icon open the dropdown
 
Line 6 is the first icon from the right. The secondary class open-right-sidebar makes this icon open the right sidebar.
 
Note: Each header icon has 2 classes, one for positioning and one that will action a menu. If you want any icon to act as a simple link, just remove the secondary class and fill in the href with your link. 
 
The secondary classes can be placed to any icon number you want, so you can make for example the last 2 icons open the sidebar by giving them the appropriate secondary classes.
 
Please be sure that the menu you are using is present on your page! Check the menus section! 
 
 
 

Sidebar Menu

Changing the sidebar logo

The sidebar logo is dependant on the sidebar color. Each sidebar has the class sidebar-DIRECTION and the secondary class sidebar-dark or sidebar-light. Toggling between these 2 will change the color of your sidebar. To change the logo of the sidebar, open menu.css and go to line 222 for the dark sidebar or line 237 for the light sidebar. After replacing the images, just as for the header logo you'll have to change the sizes of your logo on line 262 in menu.css

To activate a submenu by default. On Line 1 you must add active-item, the same must be done for line 2 and in order to have the submenu visible, you must add active-submenu to line 7. 

Do you want to change the width of the sidebars?

We made changing the sidebar width a little easier than in our last products, and it's done by javascript. Open custom.js and go to line 56. There you will find a value, currently at 250, replace that with whatever width you desire (keep in mind smaller devices that have a screen width of 320 pixels ) Now, after you replace that value, on line 56, just add 30 pixels to the value you just created, and add it there. That will compensate for the shadow effect of the sidebar so it doesn't show on the left and right of your screen.

That's it!

These don't offer a lot of screen space so all the icons need to be added in perfectly. If you use more than 6 icons ( they are not programmed in ) we'll gladly explain how, via our support page http://enableds.com/html-support/ but we're not responsible for users not seeing any menu item because it gets cut out.

The footer menu comes with the secondary class footer-menu-dark this can be replaced with footer-menu-light to change the color of the footer menu. 

The footer menu is not activated by any header or secondary class. This has to be copy and pasted to the pages you want. Placement is like all other menus, anywhere you wish except inside other menus or inside the content class. 

The footer menu is comprised of 10 items, 5 of which are only visible on tablets .These have the class footer-tablet-item. The reason behind this is again screen size. Devices like the iPhone 5S or iPhone SE have a actual screen resolution that permits a perfect 5 icon layout. Adding more would make them untappable. Simply replace the href with your link and you can also add the class active-footer-item to the ones you want to set active. 

Please note! Inside the actual footer of the page at the very end right after the copyright message you will notice a class <div class="footer-clear disabled"></div> For pages that use the footer menu please remove the disabled secondary class. Without this the footer will be cut-off due to the footer menu being fixed.

Removing unneeded features and pages

After seting your page up, and running, make sure you remove unnecessary elements from the menu.css create a backup for further referrence and delete all the classes that are not used. The menu.css has commented out chapters. Delete the entire chapter that you don't need in your page. 

If you don't use the News, Store or Blog sections, you can easily remove these from framework.css 

We serve a large array of customers and some use these pages some don't. It's best to know where to find them and remove them if you don't need them. 

Store Template Lines 4752 to 5888

News Template Lines 5930 to 6484

Blog Template Lines 6484 to 6767

Removing these elements if unnecessary will ligthen the load on your page by a few kilobytes. 

You can remove any elements you don't need from the framework.css. You'll find each page, pageapp or individual element is setup inside it's own commented chapter for easy refference.
