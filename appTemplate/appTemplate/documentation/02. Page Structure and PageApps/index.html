Secondary Classes

Secondary classes or options classes are give you the ability to further customize an element. You'll see a lot of references inside the docuemntation of adding Secondary Classes to elements, and these are the available secondary classes you can use. 

Spacing

full-bottom: adds a margin bottom to an element, that is equal to the container spacing, to keep visual hiearchy

half-bottom: adds half the total of the margin bottom used throughout containers

no-bottom: this class can be used if you are using an element within a container that has a full bottom at the end. That will result in having a full bottom added by default from both elements. To counter this, you can add no-bottom to any of the elements to keep one of them from applying the bottom. An instant example would be if you have a parahraph inside a container. You can add the class no-bottom either to the parahraph or the container and only one of them will add the bottom margin.

Text Features

You can set the text to center, left, or right, and also modify it from bold to light, italic or regular using simple classes as well

center-text makes the text centered on all resolutions

right-text makes the text right aligned on all resolutions

left-text makes the text left aligned on all resolutions

Adding -mobile or -tablet to the the above classes will apply the setting only for those devices. 

uppercase makes the text uppercased

lowercase makes the text lowercased

thin makes the text thin

bold makes the text bold ( like the default strong class )

ultrabold makes the text the maximum bold available for that font.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Content Sliders

For the content sliders we've used the awesome OWL Carousel 2. A powerhorse when it comes to smooth sliders, great transitions, and excellent performance! Although we agree that this slider uses the latest and greatest tech to improve your performance, we STRONGLY recommend that you use the least ammount of slides possible, and the least ammount of sliders possible on a page. Please remember, you may be rocking an octo-core with a great GPU in your hand, but at the end of the day, it's not made to handle the workload of a desktop. Furthermore, the more images you load, the more data is required to download them all! 

 

All sliders run on the same principle! 

You can create however many sliders you want, and duplicate their main inner div, to create more elements. It's that easy! Let's take the homepage slider for example. 

            <div class="slider-wrapper">
                <a class="next-slide home-next" href="#"><i class="fa fa-angle-right"></i></a>
                <a class="prev-slide home-prev" href="#"><i class="fa fa-angle-left"></i></a>

<div class="single-item homepage-slider"> <div> <div class="slider-caption"> <h3>Meet Bolt</h3> <p>It's fast as lightning!</p> </div> <img class="responsive-image" width="900" src="images/pictures/1.jpg" alt="img"> </div> </div> </div>

The code above generates the homepage slider.

Line 1 is the slider wrapper. Please use this to maintain and keep performance high by giving the slider an external class to be housed in it allows animations to run better

Line 2 and 3 are the slider controls. The secondary class controls the slider motion, home-next and home-prev. 

Line 4 is single-item, which defines a single item slider. This is the call for the javascript to generate the slider. The secondary class is used to style the elements inside. 

Line 5 is the actual div you'll have to duplicate to create more slides. PLEASE NOTE that if you load the slider as is, without creating at least 2 slides, it will not function. 

Pretty much easy, isn't it? Now, let's explain the JS Behind the sliders, and teach you how to update and modify certain functions. 

 

    $('.single-item').owlCarousel({
        autoplay:true,
        autoplayTimeout:4000,
        loop:true,
        margin:10,
        nav:false,
        dots:false,
        responsive:{
            0:{
                items:1
            },
            600:{
                items:1
            },
            1000:{
                items:1
            }
        }
    });
    $('.home-next').click(function() {
        $('.single-item').trigger('next.owl.carousel');
    }); 
 
    $('.home-prev').click(function() {
        $('.single-item').trigger('prev.owl.carousel');
    });

The JS for the sliders are found in custom.js, you can modfy them with great ease by changing different parameters. 

autoplay: sets the autoplay to true or false

autplayTimeout: can be replaced with values you require in miliseconds. 4000 means 4 seconds. 

loop: true or false, to create an infinite loop on the slider

nav and dots must be left to false.

The responsive parameter sets breakpoint at which you can select different resolution sizes to show different elements

The home-next and home-prev slider target the single item slider. If you have multiple sliders on the same page and wish to control them individually, you must first duplicate the entire thing shown above, and replace single-item with a new class. Then they will be indivudually controlled. Same applies for the double-item and tripple-item sliders.

Image Sliders

All the image sliders in our products are easy to use, and very, very simple to customize. 

Let's take a fast example let's say you're using a single-item slider and you want to create more slides. Here is how it's structure will look

 

<div class="single-item">
    <div class="swiper-wrapper">
        <div class="swiper-slide">
             <!--Actual Slide Content-->
        </div>
    </div>
</div>

Line 1 defines the slider for that specific page. You'll notice some pages have different sliders, for example the news page will have the news-slider. These classes are used to style elements inside the slides, so the slider class and content are important

Line 2 is the slider-wrapper. Why use this class you ask? To keep oveflow errors from occurring on Android devices. Some Androids don't play well with sliders and can make the content jump up and down. This solves it! 

Line 3 is the swiper-slide. This is the actual slide of your slider. To create more tiems, just duplicated lines 3 4 5 and replace the content inside line 4. It's that simple, and it applies to all our sliders! 

NOTE: Having a vast documentation for each and every slider would be really innefective as explaining slides isn't quite easy, we strongly recommend learning by example, meaning, we have included all sldiers in the sample pages you get with the files. Duplicate the slide that interestes you, and replace the text and image with your own! 

Yes, you can have multiple slides with the same style! You can have as many as you want. Just one BIG note! 

Remember, you're working with mobile devices! Loading 50 high resolution images in a slider can cause the page to load slowly! We recommend using tinypng.com for images ( it works with JPG too ) It will reduce your image size significantly without affecting it's quality! 

Gallery Round & Square

The difference between a round gallery and a square gallery is the aditional class. Replace round-thumbs with square-thumbs and the gallery become square thumbed! 

The gallery does not have to be inside a container, you can place it outside as well. 

<div class="gallery round-thumbs">
    <a href="images/pictures/1.jpg" class="show-gallery" title="Your caption here!"><img src="images/pictures/1s.jpg" alt="img"></a>                                  
</div>

To create more elements, simply duplicate the anchor class inside and replace the href and the img src with with your images. 

 

Gallery Justified

<div class="gallery-justified">
    <a href="IMG_LOCATION" class="show-gallery" title="Caption"><img src="IMG_LOCATION"></a>
    <a href="IMG_LOCATION" class="show-gallery" title="Caption"><img src="IMG_LOCATION"></a>
    <a href="IMG_LOCATION" class="show-gallery" title="Caption"><img src="IMG_LOCATION"></a>    
</div>

Adding the IMG_LOCATION to your images location will automatically generate a justified gallery. The images will be alinged in order of the design of these images. Using 16:9 images will create wide thumbs, using 1:1 images will use square thumbs and using 9:10 images will create tall thumbs. 

A justified gallery means the images will always be in line with each other, basically, like a mansonry design.  All the images you'll add here will automatically be alinged to a certain height. To change the height you want to display, open custom.js  and locate the code below. 

    function apply_gallery_justification(){
        var screen_widths = $(window).width();
        //MOBILE SETTINGS
        if( screen_widths < 768){ 
            $('.gallery-justified').justifiedGallery({
                rowHeight : 70,
                maxRowHeight : 370,
                margins : 5,
                fixedHeight:false
            });
        };
        // TABLET SETTINGS
        if( screen_widths > 768){
            $('.gallery-justified').justifiedGallery({
                rowHeight : 150,
                maxRowHeight : 370,
                margins : 5,
                fixedHeight:false
            });
        };
    };

The code above generates the elements for each scenarion, mobile and tablet. You can replace the rowHeight value from both to change the height for different screens.

The code is highlighted with some numbers which we'll explain exactly what each do. 

 

(1) The secondary class, portfolio-adaptive-three sets the default state of the portfolio, currently, it loads showing 3 items, if you want it to show 1 column set it to portfolio-adaptive-one and if you want it it to show 2 columns set it to portfolio-adaptive-two

(2) The buttons that change the portfolio can be selected by adding the secondary class active-adaptive to any button. This will turn it orange, showing it's activated. 

(3) Duplicating the entire portfolio-item class will generate a new item for the portfolio or gallery. In order to generate a gallery, you will only need to leave item (4) in there. Items (5), (6), (7) can be removed. This way, you will have an adaptive image gallery. To keep the portfolio aspect of the item, simply leave (5), (6), (7) in there and add your content to them.

Fullscreen Pages! Attention!

All fullscreen pages are aligned to act like apps. These MUST have the body class set to has-cover. 

<body class="has-cover">

If this is not set, the page will not function. We used this because in order to check the screen size on turnig in portrait or landscape on a tablet we need a resize handler, a function that constantly checks if the screen size changed. If this is left on all pages, the page performance will decrease, so we create an if statement, that if a page has that class, the reize handler will trigger and apply.
